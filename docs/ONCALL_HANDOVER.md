# 值班交接手册（On-call Handover）

## 1. 班前检查（接班人）

1. 确认本地可访问看板与配置页
2. 执行 `npm run ops:doctor`
3. 查看当前未恢复告警与 ACK 列表
4. 查看最近 24 小时审计日志与关键变更

## 2. 交接信息模板（交班人 -> 接班人）

- 当前告警概览（目标/级别/开始时间）
- 已 ACK 事项（负责人/备注）
- 临时静默窗口（结束时间）
- 未完成变更（配置、发布、回滚）
- 高风险目标清单（Top 5）
- 需要重点观察的指标/服务

## 3. 值班中操作约束

- 任何配置变更必须可审计（优先走 API）
- 高风险操作前先在群里声明
- 事故期间每 10~15 分钟同步状态
- 操作后补全记录（时间、动作、结果）

## 4. 班后输出

- 当班事件摘要
- 异常告警处置结论
- 未完成事项及移交 owner
- 建议优化项（阈值、通知范围、文档）
