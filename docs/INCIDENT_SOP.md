# 事故响应 SOP（Incident SOP）

## 1. 事故分级

- P1：核心业务中断 / 大面积离线 / 持续高危告警
- P2：部分功能受损 / 局部高危告警
- P3：单点异常 / 可延后处理的问题

## 2. 5 分钟内动作

1. 建立事故频道（群/会议）
2. 指定角色：指挥、排障、记录、沟通
3. 执行 `npm run ops:doctor`，收集首轮证据
4. 从看板确认影响范围（env/business/room/owner）

## 3. 30 分钟内动作

1. 明确故障类型：network / auth / api / resource
2. 使用 `/api/v1/targets/diagnose` 验证关键目标
3. 必要时进行临时止损（降级、扩容、回滚）
4. 每 10 分钟同步一次状态与下一步

## 4. 回滚策略

- 配置问题：使用配置回滚接口恢复
  - `POST /api/v1/config/rollback`
- 版本问题：回滚到上一个稳定镜像或 git ref
- 回滚后必须执行健康检查与关键路径验证

## 5. 结束条件

- `readyz` 全部恢复
- 关键告警消退或明确进入观察窗口
- 对业务方给出恢复确认
